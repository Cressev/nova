## 2. 技术目标

Nova v1.0 的目标是实现一个本地优先的 Web Agent 网关。用户通过浏览器和 Agent 交互，CLI 只作为启动、配置和诊断入口。

v1.0 技术目标：

- 提供本地 Web UI。
- 提供 Python FastAPI 后端网关。
- 实现基础 Agent Runtime。
- 实现项目上下文加载。
- 实现文件、搜索、shell、git diff 等基础工具。
- 实现权限审批。
- 实现本地 Markdown 项目记忆。
- 实现 JSONL session 和 trace。
- 集成 Langfuse 作为开发调试可观测工具。
- 在前端提供用户可理解的任务进度、工具调用、审批和 trace 摘要。

## 3. 总体架构

```text
浏览器 Web UI
  -> FastAPI 本地网关
     -> Agent Runtime
        -> 上下文加载器
        -> 模型 Provider 适配器
        -> 工具注册表
        -> 权限引擎
        -> Trace 记录器
        -> 记忆更新器
        -> Langfuse Reporter
```

## 4. 技术栈

### 4.1 前端

| 模块 | 选型 |
|---|---|
| 框架 | React |
| 构建 | Vite |
| 样式 | Tailwind CSS |
| 组件 | shadcn/ui |
| 状态 | Zustand 或 TanStack Query |
| 实时通信 | WebSocket，必要时补 SSE |

前端承担最终用户交互，不依赖 Langfuse 给用户展示执行过程。

### 4.2 后端

| 模块 | 选型 |
|---|---|
| 语言 | Python |
| Web 框架 | FastAPI |
| 实时通信 | WebSocket |
| 异步运行 | asyncio / anyio |
| 配置 | TOML |
| 数据校验 | Pydantic |
| 命令执行 | asyncio subprocess 或 anyio subprocess |
| 文件处理 | pathlib |
| 搜索 | 调用 ripgrep |
| 日志 | JSONL + structlog 可选 |

后端优先 Python，便于学习、调试和后续扩展。

### 4.3 可观测

| 类型 | 选型 |
|---|---|
| 开发调试可观测 | Langfuse |
| 本地 trace | JSONL |
| 用户侧可观测 | Nova Web UI |

Langfuse 不作为用户界面，只用于开发者查看 trace、token、cost、latency、prompt、eval 和错误。

## 5. 核心模块设计

### 5.1 FastAPI Gateway

职责：

- 提供 Web API。
- 提供 WebSocket 实时事件流。
- 管理 session。
- 接收用户输入。
- 向前端推送 Agent 运行事件。
- 处理审批请求和审批结果。

候选接口：

```text
GET  /api/health
GET  /api/sessions
POST /api/sessions
GET  /api/sessions/{session_id}
POST /api/sessions/{session_id}/messages
WS   /api/sessions/{session_id}/stream
POST /api/approvals/{approval_id}/approve
POST /api/approvals/{approval_id}/deny
GET  /api/memory
GET  /api/traces/{trace_id}
```

### 5.2 Web UI

核心页面：

- 会话列表。
- Agent 对话页面。
- 工具调用时间线。
- 审批弹窗。
- 文件 diff 预览。
- 任务结果面板。
- 项目记忆面板。
- 本地 trace 摘要页。

用户侧可观测展示：

- 当前阶段：理解需求、读取上下文、执行工具、等待审批、总结结果。
- 工具调用卡片：工具名、状态、摘要、耗时、风险等级。
- Shell 命令卡片：命令、目的、风险说明、输出摘要。
- 文件修改卡片：路径、变更摘要、diff。
- 失败卡片：失败原因、重试建议。

### 5.3 Agent Runtime

职责：

- 管理一次 Agent run。
- 构造模型输入。
- 解析模型输出。
- 触发工具调用。
- 接收工具结果。
- 判断是否继续循环。
- 生成最终回答。

运行循环：

```text
用户输入
  -> 加载上下文
  -> 调用模型
  -> 如果模型请求工具：权限检查
  -> 执行工具
  -> 记录 trace
  -> 返回工具结果给模型
  -> 直到模型输出最终结果
```

### 5.4 Context Loader

启动任务前读取：

- `AGENTS.md`
- `CLAUDE.md`
- `CURRENT.md`
- `PROGRESS.md`
- `log.md` 末尾
- `user-queries.md` 末尾
- 相关目录 README
- 用户显式引用的文件

约束：

- 不把大文件完整塞入上下文。
- 对长文件做摘要或片段读取。
- 不读取敏感文件，除非用户确认。

### 5.5 Model Provider Adapter

v1.0 优先支持：

- OpenAI-compatible endpoint。

v1.1 或后续支持：

- Anthropic。
- Kimi。
- OpenRouter。
- 本地模型服务。

接口抽象：

```python
class ModelProvider:
    async def stream(self, request: ModelRequest) -> AsyncIterator[ModelEvent]:
        ...
```

### 5.6 Tool Registry

v1.0 工具：

| 工具 | 用途 | 风险 |
|---|---|---|
| `read_file` | 读取文件 | 低到中 |
| `list_dir` | 查看目录 | 低 |
| `search_rg` | 搜索项目 | 低 |
| `apply_patch` | 修改文件 | 中 |
| `shell` | 执行命令 | 中到高 |
| `git_diff` | 查看 diff | 低 |
| `update_memory` | 更新记忆文件 | 中 |

工具定义应使用 Pydantic schema，确保参数可校验。

### 5.7 Permission Engine

权限目标：

- 在工具执行前判断是否允许。
- 高风险操作生成审批请求。
- 将审批请求推送到前端。
- 等待用户批准或拒绝。

权限策略：

| 操作 | 默认策略 |
|---|---|
| 读取 workspace 普通文件 | 允许 |
| 读取敏感文件 | 拒绝或询问 |
| 写 workspace 文件 | 询问或按策略允许 |
| workspace 外写入 | 拒绝 |
| shell 只读命令 | 询问或允许 |
| shell 破坏性命令 | 必须询问 |
| 网络访问 | 默认询问 |

敏感路径示例：

- `.env`
- `.ssh/`
- token 文件
- key 文件
- 凭证配置

### 5.8 Trace Recorder

本地 trace 使用 JSONL：

```json
{"type":"run_start","run_id":"...","session_id":"...","timestamp":"..."}
{"type":"tool_call","tool":"read_file","input_summary":"...","timestamp":"..."}
{"type":"approval","approval_id":"...","decision":"approved","timestamp":"..."}
{"type":"tool_result","tool":"read_file","status":"ok","output_summary":"..."}
{"type":"run_end","status":"completed","timestamp":"..."}
```

trace 原则：

- 本地先落盘。
- 再异步上报 Langfuse。
- 不保存敏感明文。
- 长输出只保存摘要。

### 5.9 Langfuse Reporter

Langfuse 用途：

- 开发调试。
- 观察模型耗时。
- 观察 token 和成本。
- 分析工具调用链路。
- 管理 prompt 版本。
- 后续做 eval。

Langfuse 不做：

- 不作为用户界面。
- 不承担审批。
- 不作为唯一 trace 存储。
- 不保存未脱敏敏感内容。

### 5.10 Memory Updater

更新目标：

- `CURRENT.md`
- `PROGRESS.md`
- `log.md`
- `TODOList.md`
- `user-queries.md`
- 必要时更新 `AGENTS.md`

约束：

- 只有 durable 状态才写入。
- 用户偏好和硬规则写入 `AGENTS.md`。
- 产品研发文档正文只有用户明确要求才写。

## 9. 前后端事件协议

WebSocket 事件类型：

| 事件 | 方向 | 用途 |
|---|---|---|
| `run_started` | 后端 -> 前端 | 任务开始 |
| `assistant_delta` | 后端 -> 前端 | 模型流式输出 |
| `tool_call_started` | 后端 -> 前端 | 工具调用开始 |
| `tool_call_completed` | 后端 -> 前端 | 工具调用完成 |
| `approval_requested` | 后端 -> 前端 | 请求用户审批 |
| `approval_resolved` | 前端 -> 后端 | 用户批准或拒绝 |
| `diff_ready` | 后端 -> 前端 | 文件 diff 可查看 |
| `run_completed` | 后端 -> 前端 | 任务完成 |
| `run_failed` | 后端 -> 前端 | 任务失败 |

## 10. 用户侧可观测设计

用户侧可观测不等于 Langfuse。用户需要看到的是可理解、可操作的信息。

前端展示：

- 当前任务阶段。
- 当前执行中的工具。
- 工具调用目的。
- 工具调用结果摘要。
- 是否需要用户审批。
- 文件修改 diff。
- 错误原因。
- 最终总结。

不展示：

- 原始 prompt 全量细节。
- token 细节。
- 模型内部调试信息。
- 未脱敏 trace。

## 11. 安全设计

### 11.1 审批优先

高风险工具调用进入审批队列，前端弹窗展示：

- 工具名。
- 操作目标。
- 风险说明。
- Agent 执行理由。
- 批准/拒绝按钮。

### 11.2 敏感信息保护

- `.env` 默认敏感。
- key/token 文件默认敏感。
- trace 和 Langfuse 上报前做脱敏。
- 长输出不直接进入长期记忆。

### 11.3 文件安全

- workspace 外写入默认拒绝。
- 文件覆盖前检查是否存在。
- 修改后生成 diff。

## 12. v1.0 验收标准

- 可以运行 `nova serve` 启动后端网关。
- 浏览器可以打开 Nova Web UI。
- 用户可以发送任务。
- Agent 能读取项目记忆。
- Agent 能读取文件、搜索文件、执行 shell。
- 高风险工具调用能在前端审批。
- 工具调用过程能在前端以时间线展示。
- trace 能写入本地 JSONL。
- Langfuse 配置存在时能上报 trace。
- Langfuse 不配置时，系统仍能正常运行。
- 需求文档和技术文档保持中文。
