# 产品研发文档集

## 用途

本目录用于存放 Nova 的产品研发文档。

这里是文档容器，不是自动生成内容的地方。除 README 维护说明外，普通产研文档默认保持空白；只有用户明确要求写某份文档时，Agent 才能写入正文。

## 目录安排

```text
产品研发文档集/
├── 预研文档集/
│   ├── README.md
│   ├── 20260603_需求文档.md
│   ├── 20260603_头脑风暴.md
│   ├── 20260603_技术调研.md
│   └── 20260603_个人开发Agent核心功能详细说明和实现方案文档.md
├── v1.0/
│   ├── README.md
│   ├── 20260603_功能开发清单.md
│   ├── 20260603_v1.0发布文档.md
│   └── 20260603_v1.0技术文档.md
└── v1.1/
    ├── README.md
    ├── 20260603_v1.1功能规划.md
    ├── 20260603_v1.1技术方案.md
    └── 20260603_v1.1发布文档.md
```

## 维护规则

- README 用于说明目录用途、文件放置规则和维护方式。
- 普通产研文档只在用户明确要求时写入正文。
- 用户没有明确要求写正文时，只能创建空白文档或维护 README。
- 不得主动补写、扩写、润色普通产研文档。
- 不得删除历史文档，除非用户明确要求。
- 新增文档时，文件名使用实际创建日期，格式为 `YYYYMMDD_主题.md`。
- 新增版本目录时，目录名使用 `v主版本.次版本`，例如 `v1.2`。

## 文档分类

- `预研文档集/`：需求、头脑风暴、技术调研、具体功能方案。
- `v1.0/`：v1.0 的功能清单、技术文档、发布文档。
- `v1.1/`：v1.1 的规划、技术方案、发布文档。
- 后续版本按同样规则新增目录。

## 写入流程

1. 用户明确指定要写哪份文档。
2. Agent 读取对应目录 README 和相关上下文。
3. Agent 确认文档所属阶段或版本。
4. Agent 写入或更新指定文档。
5. 如新增文件，同步更新对应目录 README 的文件清单。
