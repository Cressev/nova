# 用户请求入口工作流模板

`user-queries.md` 是用户任务入口文档，也是 durable source of truth。

## 核心规则

1. 收到新请求时，立即追加一个 `---` 分隔的新查询块。
2. 粘贴用户原话，不重写、不润色、不合并、不删除、不重排、不纠正。
3. 运行 `date '+%y/%m/%d-%H:%M:%S %Z'`。
4. 在开始实质工作前立即追加 `[Recieve:<时间戳>]`。
5. 所有工作完成并更新 durable 文件后，再追加 `[Done:<时间戳>]`。
6. 如果本次任务做了 git 提交，则在 Done 同一行追加 commit hash。

## 禁止事项

- 不得事后补写 Recieve。
- 不得跳过 Done。
- 不得合并多个用户请求。
- 不得编辑历史任务块。
